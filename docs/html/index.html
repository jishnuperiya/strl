Sentinex

Overview

Sentinex is a modern C++20 sensor fusion and simulation suite designed for robust vehicle state estimation and system-level validation.

The project focuses on deterministic, testable, and extensible estimation pipelines suitable for automotive, robotics, and simulation-driven development workflows.

Design Goals

  • Strongly typed state and measurement representations
  • Clear separation between models, filters, and data sources
  • Deterministic simulation and replay of sensor data
  • Emphasis on correctness, observability, and numerical stability
  • Modern C++20 design with minimal runtime overhead

Core Features

  • Vehicle state estimation using Kalman filtering techniques (linear KF, EKF, extensible to UKF)
  • Explicit process and measurement models
  • Sensor abstraction for simulation and replay
  • System-level validation utilities
  • Well-documented APIs using Doxygen

Architecture

Sentinex is structured around a small set of orthogonal components:

  • State representations (e.g., VehicleState)
  • Dynamic models (process and motion models)
  • Measurement models (sensor-specific observation models)
  • Filters (Kalman Filter variants)
  • Simulation and validation tools

Headers define the public API and semantic contracts, while implementation files focus on numerical methods and algorithms.

Coding Standards

  • C++20
  • Header-only documentation using Doxygen
  • No hidden ownership or implicit state
  • Explicit units and coordinate frame conventions

Intended Audience

Sentinex is intended for:

  • Engineers working on vehicle state estimation
  • Simulation and validation engineers
  • Researchers and students studying sensor fusion
  • Developers interested in modern C++ estimation architectures
